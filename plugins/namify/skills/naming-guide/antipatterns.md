# 안티패턴 (피해야 할 것)

## 요약

| 안티패턴 | 문제 | 예시 |
|----------|------|------|
| 메타포 혼재 | 이미지 혼란 | Capture + Cultivate + Corpus |
| UX에 강한 메타포 | 학습 곡선 상승 | ore/, flux/, smelting/ |
| 브랜드 없는 직관 | 차별화 실패 | Notes, Files, Folders |
| 설명이 필요한 이름 | 첫인상 실패 | Corpus |
| 문화적 맹점 | 타겟 이탈 | 술 메타포 → 미성년자 제외 |

---

## 1. 메타포 혼재

### 문제

하나의 제품에 여러 메타포를 섞으면 이미지가 혼란스러워짐.

### 나쁜 예시

```
Capture (사냥) + Cultivate (농업) + Corpus (학술)
→ 사냥꾼? 농부? 학자? 정체성 불명
```

### 좋은 예시

```
Gemify: 광물 메타포 일관 유지
rough → refine → gem (모두 광물)
```

---

## 2. UX에 강한 메타포

### 문제

폴더명/기능명에 강한 메타포를 사용하면 학습 곡선이 올라감.

### 나쁜 예시

```
~/.gemify/
├── ore/           # 원석? 이게 뭐야?
├── flux/          # 용제? 뭘 넣어야 해?
└── smelting/      # 제련소? 어디로 가야 해?
```

### 좋은 예시

```
~/.gemify/
├── inbox/         # 아, 받은 편지함처럼
├── drafts/        # 아, 초안
└── library/       # 아, 정리된 지식
```

---

## 3. 브랜드 없는 직관

### 문제

모든 것이 너무 평범하면 차별화 실패.

### 나쁜 예시

```
제품명: Notes
폴더: Files, Documents, Archive
→ 수백 개의 유사 제품과 구별 안 됨
```

### 좋은 예시

```
제품명: Gemify (브랜드: 보석 메타포)
폴더: inbox/drafts/library (UX: 직관적)
태그라인: "Turn your thoughts into gems"
```

---

## 4. 설명이 필요한 이름

### 문제

이름을 듣고 "이게 뭐야?"라고 물어야 하면 실패.

### 나쁜 예시

```
Corpus → "코퍼스? 그게 뭐야?"
→ 일반인에게 설명 필요
```

### 좋은 예시

```
Gemify → "보석 + ~화" → 어느 정도 짐작 가능
→ 태그라인으로 완성: "Turn your thoughts into gems"
```

---

## 5. 문화적 맹점

### 문제

특정 문화권에서 부정적이거나 제한적인 의미.

### 나쁜 예시

```
Distillery (증류소) → 술 메타포
→ 미성년자 타겟 제외
→ 특정 문화권에서 부정적
```

### 좋은 예시

```
Gemify (보석) → 모든 연령 OK
→ Minecraft 세대에게 친숙
→ 성별 중립
```

- [ ] 설명 없이 대략 짐작 가능한가?
