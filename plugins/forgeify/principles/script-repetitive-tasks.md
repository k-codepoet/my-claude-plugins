## 핵심

> **반복되는 정형화된 작업은 shell script로 만들어야 한다.**

Claude가 매번 동일한 bash 명령어를 수동으로 작성하면:
- 실수 가능성 증가
- 일관성 없는 결과
- 컨텍스트 낭비

## 판단 기준

| 조건 | 스크립트화 |
|------|-----------|
| 동일한 로직이 반복됨 | ✅ 필수 |
| 입력/출력이 정형화됨 | ✅ 필수 |
| 매번 다른 판단이 필요 | ❌ 불필요 |

## 예시

### 스크립트화 해야 하는 경우

```bash
# 플러그인 검증 - 항상 같은 항목을 체크
scripts/validate-plugin.sh {plugin-path}

# 빌드/테스트 - 정해진 순서로 실행
scripts/build.sh
```

### 스크립트화 불필요한 경우

```bash
# 탐색적 검색 - 매번 다른 패턴
grep "something" ...

# 맥락에 따른 판단 - AI가 해야 할 일
```

## 스크립트 작성 원칙

1. **Exit code로 결과 반환** (0: 성공, 1: 실패, 2: 입력 오류)
2. **명확한 출력 형식** (파싱 가능하거나 사람이 읽기 쉽게)
3. **scripts/ 디렉토리에 저장**
4. **실행 권한 부여** (chmod +x)

## forgeify 적용

forgeify 스킬 실행 시 이 원칙을 참조:
- 정형화된 검증 로직 발견 시 → 스크립트화 검토
- 기존 스크립트 있으면 → 스크립트 우선 실행
