# 밀도 기반 파이프라인 (Density-Based Pipeline)

## 핵심 사상

지식은 **정제 단계**를 거칠수록 밀도가 높아진다.

```
inbox (원석) → draft (다듬기) → library (정제) → view (서사)
   낮음  ─────────────────────────────────────────▶  높음
```

## 왜 단계를 건너뛰면 안 되는가

| 건너뛴 단계 | 결과 |
|-------------|------|
| inbox 생략 | 원본 맥락 유실 |
| draft 생략 | 정제되지 않은 지식 저장 → 나중에 재작업 |
| library 생략 | 재사용 불가능한 일회성 문서 |

## 밀도 평가 기준

| 수준 | 특징 | 질문 예시 |
|------|------|----------|
| inbox | 생각/재료 상태 | "뭔가 있는데 정리 안 됨" |
| draft | 아이디어 있지만 정제 필요 | "핵심이 뭐야?" |
| library | 단일 주제로 정제됨 | "어떤 type?" |
| view | 서사가 완성됨 | "누구에게, 무엇을?" |

## 역방향 작업 (retro/triage)

이미 완료된 작업을 기록할 때도 밀도를 평가하여 적절한 단계부터 시작:
- 밀도 높음 → library/view로 직행 가능
- 밀도 낮음 → inbox부터 채워나가기 제안

## 밀도 평가 체크리스트

**library 수준 (바로 저장 가능)**:
- [ ] 단일 주제로 명확히 정의됨
- [ ] 핵심 내용이 정제됨
- [ ] type 분류가 자명함

**draft 수준 (다듬기 필요)**:
- [ ] 아이디어가 있지만 산만함
- [ ] 여러 주제가 섞여 있음
- [ ] 핵심 추출이 필요함

**inbox 수준 (정리 필요)**:
- [ ] 생각/재료 상태
- [ ] 구조화되지 않음
- [ ] 맥락 없이 단편적
